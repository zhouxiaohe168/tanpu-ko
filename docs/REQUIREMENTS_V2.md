# 探铺 V2 需求文档

## 需求来源
用户反馈 + 商业模式优化

## 核心需求

### 1. 用户权限控制（严格版）

**当前问题**：
- 未登录用户可以看到所有数据
- 没有真正的收费门槛

**目标状态**：
- 未注册/未登录用户：完全不能查询，点击任何查询按钮都跳转登录页
- 已登录免费用户：仅限1次免费查询机会
- 已付费用户：根据套餐享受对应次数

**权限矩阵**：
| 用户类型 | 查询次数 | 查看结果 | AI研判 | 导出 |
|----------|----------|----------|--------|------|
| 未登录 | 0 | 不可 | 不可 | 不可 |
| 免费用户 | 1次/永久 | 完整1次 | 不可 | 不可 |
| 单次付费 | 1次 | 完整 | 1次 | 不可 |
| 月卡会员 | 50次/月 | 完整 | 50次/月 | 可 |
| 年卡会员 | 无限 | 完整 | 无限 | 可 |

### 2. 行业和品牌选择（简化版）

**当前问题**：
- 行业太多，数据不全
- 竞品选择不够灵活

**目标状态**：
- 行业暂时只保留：奶茶饮品
- 品牌选择：奶茶行业 TOP 品牌（蜜雪冰城、古茗、茶百道、沪上阿姨、书亦烧仙草、霸王茶姬、喜茶、奈雪的茶、CoCo都可、一点点）

**竞品选择逻辑**：
```
竞品类型: [本行业] [跨行业]

如果选择"本行业"：
  → 显示奶茶行业 TOP 品牌多选框

如果选择"跨行业"：
  → 先选择行业（咖啡/快餐/便利店等）
  → 再显示该行业 TOP 品牌多选框
```

### 3. 查询流程改造

**改造前**：
```
用户输入 → 直接显示结果
```

**改造后**：
```
用户输入 → 检查登录状态
  ├─ 未登录 → 跳转登录页（带回调URL）
  └─ 已登录 → 检查剩余次数
       ├─ 有次数 → 扣减1次 → 显示结果
       └─ 无次数 → 显示付费墙
```

## 技术实现要点

### 数据库改造
1. `profiles` 表增加字段：
   - `free_query_used` BOOLEAN DEFAULT FALSE -- 免费次数是否已用
   - `query_count` INTEGER DEFAULT 0 -- 已使用查询次数

2. 每次查询时：
   - 先检查用户权限
   - 扣减次数（记录到 queries 表）
   - 返回结果

### API 改造
1. `/api/query` 增加权限校验中间件
2. 返回结果前检查用户权限
3. 无权限返回 401/403

### 前端改造
1. 查询按钮点击前检查登录状态
2. 登录状态用 SWR 缓存
3. 付费墙组件优化
